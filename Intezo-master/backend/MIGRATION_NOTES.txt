MONGOOSE TO SEQUELIZE MIGRATION NOTES - doctorController.js
================================================================

MONGOOSE METHODS FOUND AND THEIR SEQUELIZE REPLACEMENTS:
---------------------------------------------------------

1. Doctor.find() - Query multiple documents
   MONGOOSE: Doctor.find({ 'clinics.clinic': clinicId, 'clinics.isActive': true })
   SEQUELIZE: Doctor.findAll({ where: { /* conditions */ } })
   
   For JSONB queries:
   SEQUELIZE: Doctor.findAll({ 
     where: sequelize.literal("clinics @> '[{\"clinic\": \"" + clinicId + "\", \"isActive\": true}]'")
   })

2. .select() - Select specific fields
   MONGOOSE: .select('-password')
   SEQUELIZE: { attributes: { exclude: ['password'] } }
   OR
   SEQUELIZE: { attributes: ['name', 'email', 'phone'] }

3. .populate() - Join related data
   MONGOOSE: .populate('clinics.clinic', 'name')
   SEQUELIZE: { include: [{ model: Clinic, attributes: ['name'] }] }
   NOTE: For JSONB fields, population doesn't work - need separate queries

4. Doctor.findOne() - Query single document
   MONGOOSE: Doctor.findOne({ _id: req.params.id, clinic: req.clinic._id })
   SEQUELIZE: Doctor.findOne({ where: { id: req.params.id, clinicId: req.clinic.id } })

5. new Doctor() + save() - Create new document
   MONGOOSE: 
     const doctor = new Doctor({ name, specialty });
     await doctor.save();
   SEQUELIZE: 
     const doctor = await Doctor.create({ name, specialty });

6. Doctor.findOneAndUpdate() - Find and update
   MONGOOSE: Doctor.findOneAndUpdate(
     { _id: req.params.id, clinic: req.clinic._id },
     req.body,
     { new: true, runValidators: true }
   )
   SEQUELIZE: 
     const [rowsUpdated, [doctor]] = await Doctor.update(
       req.body,
       { where: { id: req.params.id, clinicId: req.clinic.id }, returning: true }
     );

7. Doctor.findOneAndDelete() - Find and delete
   MONGOOSE: Doctor.findOneAndDelete({ _id: req.params.id, clinic: req.clinic._id })
   SEQUELIZE: 
     const doctor = await Doctor.findOne({ where: { id: req.params.id, clinicId: req.clinic.id } });
     if (doctor) await doctor.destroy();

8. Doctor.updateOne() - Update without returning document
   MONGOOSE: Doctor.updateOne(
     { _id: doctorId, 'clinics.clinic': clinicId },
     { $set: { 'clinics.$.isAvailable': isAvailable } }
   )
   SEQUELIZE: 
     await Doctor.update(
       { /* updates */ },
       { where: { id: doctorId } }
     );
   NOTE: For JSONB array updates, use sequelize.literal() or raw queries

9. Doctor.findById() - Find by primary key
   MONGOOSE: Doctor.findById(doctorId)
   SEQUELIZE: Doctor.findByPk(doctorId)

10. Queue.find() - Query with conditions
    MONGOOSE: Queue.find({ doctor: doctorId, status: 'waiting' })
    SEQUELIZE: Queue.findAll({ where: { doctorId, status: 'waiting' } })

11. .sort() - Sort results
    MONGOOSE: .sort('number') or .sort({ name: 1 })
    SEQUELIZE: { order: [['number', 'ASC']] } or { order: [['name', 'ASC']] }

12. .limit() - Limit results
    MONGOOSE: .limit(10)
    SEQUELIZE: { limit: 10 }

13. Queue.countDocuments() - Count documents
    MONGOOSE: Queue.countDocuments({ doctor: doctorId, status: 'served' })
    SEQUELIZE: Queue.count({ where: { doctorId, status: 'served' } })

14. Doctor.findByIdAndUpdate() - Find by ID and update
    MONGOOSE: Doctor.findByIdAndUpdate(
      req.doctor._id,
      { name, phone },
      { new: true, runValidators: true }
    )
    SEQUELIZE:
      const doctor = await Doctor.findByPk(req.doctor.id);
      await doctor.update({ name, phone });
      // OR
      const [rowsUpdated, [doctor]] = await Doctor.update(
        { name, phone },
        { where: { id: req.doctor.id }, returning: true }
      );

15. Doctor.aggregate() - Aggregation pipeline
    MONGOOSE: Doctor.aggregate([
      { $match: { 'clinics.clinic': clinicId } },
      { $project: { name: 1, specialties: 1 } }
    ])
    SEQUELIZE: Use raw queries or complex findAll with attributes and where
    NOTE: Sequelize doesn't have direct aggregation pipeline equivalent
    Use sequelize.query() for complex aggregations

16. mongoose.Types.ObjectId() - Convert to ObjectId
    MONGOOSE: new mongoose.Types.ObjectId(clinicId)
    SEQUELIZE: Not needed - use UUID strings directly

17. doctor.save() - Save instance changes
    MONGOOSE: 
      doctor.profilePhoto = photoUrl;
      await doctor.save();
    SEQUELIZE:
      doctor.profilePhoto = photoUrl;
      await doctor.save();
      // OR
      await doctor.update({ profilePhoto: photoUrl });

18. $ne operator - Not equal
    MONGOOSE: { 'clinics.clinic': { $ne: clinicId } }
    SEQUELIZE: { where: { clinicId: { [Op.ne]: clinicId } } }
    NOTE: Need to import Op from sequelize

19. Array operations (push)
    MONGOOSE: 
      doctor.clinics.push({ clinic: clinicId, consultationFee });
      await doctor.save();
    SEQUELIZE: For JSONB arrays, need to use sequelize.fn or raw SQL
      await doctor.update({
        clinics: sequelize.fn('array_append', sequelize.col('clinics'), newClinic)
      });

20. Date range queries
    MONGOOSE: { servedAt: { $gte: today, $lt: tomorrow } }
    SEQUELIZE: { 
      where: { 
        servedAt: { 
          [Op.gte]: today, 
          [Op.lt]: tomorrow 
        } 
      } 
    }

FIELD NAME CHANGES:
-------------------
_id → id (Sequelize uses 'id' by default for UUID primary keys)
__v → Not needed (Sequelize doesn't use version keys)

IMPORTANT NOTES:
----------------
1. Import Op from sequelize for operators:
   import { Op } from 'sequelize';

2. JSONB queries in PostgreSQL are different from MongoDB nested queries:
   - Use sequelize.literal() for complex JSONB queries
   - Use @>, @<, ->, ->> operators for JSONB operations

3. Mongoose hooks (pre/post save) → Sequelize hooks (beforeSave, afterSave)

4. Populate doesn't work with JSONB fields - need separate queries or proper relations

5. Aggregation pipelines need to be rewritten as SQL queries or complex Sequelize queries

6. Array operations on JSONB fields require PostgreSQL array functions

7. Always use transactions for multi-step operations in Sequelize

8. Sequelize validation is different - define in model schema, not in queries

SUMMARY OF METHODS USED IN doctorController.js:
------------------------------------------------
- find() → findAll()
- findOne() → findOne()
- findById() → findByPk()
- findOneAndUpdate() → update() with returning: true
- findOneAndDelete() → findOne() + destroy()
- findByIdAndUpdate() → findByPk() + update()
- updateOne() → update()
- countDocuments() → count()
- aggregate() → raw queries or complex findAll
- new Model() + save() → create()
- instance.save() → instance.save() or instance.update()
- select() → attributes option
- populate() → include option
- sort() → order option
- limit() → limit option
